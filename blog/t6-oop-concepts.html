Scoping

July 5th, 2015

In getting into Ruby I had no idea variable scoping was important(or even existed really!). But over the course of the past few weeks I've had to learn quickly that scoping is thing, and is quite important. In this post I'll go into what your options are with scoping, and when you should use each.

For the Different types of scoping we have global(begins with $), instance(begins with @), local(begins with a-z), constant(begins with A-Z), or class (begins with @@).

Local variables are pretty straight forward - they can be accessed locally within the code they are declared. So - local variable can't be accessed outside of the method or loop in which they are created.

Global variables are also very straightforward - they can be accessed anywhere within the Ruby program in which they are created.

Class variables are variables that are shared among all instances of a class - this means that if something changes this variable - it will change it for all instances of the class you have.

Instance variables are like class variables except that they are specific to each individual instance of a class - if you change the value in one instance - it isn't going to change the value in all instances - just that one instance.

Constant variables are just that - constant. They shouldn't be changed once they are created. Constant variables declared outside of a class or variable are considered global - while those created within a class etc. are available throughout whatever they are created within.
